Responsible Use

Everything PhantomRecon gathers is passive, publicly available OSINT — WHOIS records, DNS, certificate transparency logs, and content already published on the target's own public pages. This is the same standard reconnaissance phase used in authorized penetration testing and bug bounty programs. It does not exploit, brute-force, or access anything non-public. Always have authorization before conducting reconnaissance against a domain you don't own.

WHOIS

Raw WHOIS protocol (RFC 3912) implemented from scratch — TCP port 43. Follows the standard referral chain: IANA bootstrap → TLD registry → registrar (for gTLDs like .com/.net where Verisign is the registry but the registrar holds full details).

DNS Records

Custom DNS client built at the packet level (RFC 1035) — no external DNS library. Queries a public resolver directly (stub resolver pattern) and parses responses including name compression.

A / AAAAIPv4/IPv6 address(es) the domain points to

Subdomain Enumeration

Passive: Certificate Transparency logs via crt.sh. Every publicly-trusted TLS certificate issued since ~2018 is logged publicly — searching these logs reveals every subdomain anyone has ever requested a certificate for, including forgotten staging/dev subdomains, without sending a single packet to the target.

Active: DNS resolution of ~70 common subdomain prefixes (www, mail, admin, staging, vpn, etc.) — catches subdomains that exist but never got a public certificate.

HTTP Fingerprinting

Detects technology stack from response headers, HTML content, cookies, and meta generator tags — the same passive signals "view source" reveals. Also checks for well-known files (robots.txt, sitemap.xml, security.txt) and standard security headers.

Email Discovery

Extracts email addresses that already appear on the target's own public pages (homepage, /contact, /about, /team) — including simple de-obfuscation of "user [at] domain [dot] com" patterns. Never guesses or generates likely addresses — only reports what's actually published.

IP Intelligence

Geolocation and hosting organization/ASN via ip-api.com (free tier), plus reverse DNS (PTR record) lookup via our own DNS module — so hostname info is still available even if the geolocation API is unreachable.
